## VERSI PANJANG (rekomendasi untuk pesan pertama kali)

```
Assalamu'alaikum wr. wb.

Mohon bantuan njenengan testing aplikasi *Ammu Online* sebelum kami rilis resmi ke Play Store. Berikut panduan singkatnya:

══════════════════════════
*🔧 LANGKAH INSTALL*
══════════════════════════

1. Hapus dulu APK Ammu lama (kalau sudah pernah install).
2. Download APK baru dari link yang saya kirim.
3. Install — kalau muncul peringatan "sumber tidak dikenal", izinkan saja (sumbernya saya pribadi).
4. Buka aplikasi, login pakai akun guru njenengan.

══════════════════════════
*✅ YANG PERLU DICOBA*
══════════════════════════

*A. LOGIN & MASUK*
□ Login pertama kali — apakah lancar?
□ Tutup aplikasi, buka lagi — masih login tidak? (harusnya tidak logout)
□ Logout manual lalu login lagi — lancar?

*B. HALAMAN BERANDA*
□ Jam digital + tanggal Hijri Arab tampil benar
□ Kalender pendidikan (agenda hari ini & terdekat) muncul
□ Postingan dari admin terbaca
□ Tarik layar ke bawah (pull-to-refresh) — apakah ada efek refresh?

*C. PROFIL SAYA (sidebar atas)*
□ Ganti foto profil — upload berhasil?
□ Upload Tanda Tangan Digital — preview muncul?
□ Ubah Nomor WhatsApp — tersimpan?
□ Ganti Username — tersimpan?
□ Ganti Kata Sandi — bisa login pakai password baru?
□ Aktifkan Notifikasi — toast "Notifikasi Aktif" muncul?
□ Tautkan Akun Google — connect berhasil?

⚠️ *Yang penting:* saat tap salah satu pengaturan di atas, *hanya* form yang dipilih yang muncul (bukan semua field). Kalau muncul gak sesuai, foto/sandi/dll. tercampur — tolong screenshot.

*D. ABSENSI SANTRI*
□ Buka halaman Absensi Santri Sekolah/Diniyah
□ Input absen hari ini (Hadir/Sakit/Izin/Alpa) — tersimpan?
□ Ganti tanggal ke kemarin — data muncul?
□ Filter per kelas berfungsi?
□ Tombol Ekspor PDF — file ke-download? Buka file manager → folder *Documents/AmmuOnline*
□ Tombol Ekspor Excel — file ada KOP "Pondok Pesantren Mambaul Ulum" + logo?

*E. ABSENSI GURU (FINGERPRINT/MANUAL)*
□ Cek riwayat absen pribadi — data terbaca?
□ Rekap absen bulanan — ekspor PDF dan Excel jalan?

*F. RAPOR SANTRI*
□ Input nilai per mata pelajaran — tersimpan?
□ Input kepribadian (kelakuan/kerajinan/kebersihan)?
□ Input absensi rapor (S/I/A)?
□ Cetak Rapor (PDF) — preview muncul, bisa simpan/share?
□ Catatan: layout PDF rapor mungkin masih ada whitespace — itu sudah kami catat, akan diperbaiki di update Windows-version

*G. NOTIFIKASI*
□ Notifikasi pos baru dari admin — muncul di status bar HP?
□ Tap notifikasi — buka halaman terkait?

*H. TAMPILAN UMUM*
□ Header "Ammu Online" di atas tidak tertutup status bar HP?
□ Sidebar (menu kiri) tampil rapi, bisa scroll?
□ Mode gelap (toggle di pojok kanan atas) — bekerja semua halaman?
□ Tombol kembali HP — kembali ke halaman sebelumnya (bukan langsung keluar)?

══════════════════════════
*📝 CARA LAPOR BUG*
══════════════════════════

Kalau ada yang tidak normal, japri saya pesan dengan format:

```

## CATATAN INTERNAL KYAI (jangan dikirim ke guru)

**Yang sengaja TIDAK diminta tes ke guru:**

- Master Data CRUD (santri/guru/lembaga) — hanya admin utama bisa, biasanya cuma kyai sendiri yang punya akses
- Audit Log + Audit Data Legacy — admin-only, fitur teknikal
- Pengaturan Sistem (printer LX-310, jam absen range) — admin-only
- Mode iOS / iPad PWA — APK ini Android only

**Yang kyai sendiri perlu tes (sebagai admin utama):**

1. CRUD Master Data — tambah/edit/hapus santri, guru, lembaga, kelas
2. Kelola Postingan — buat, edit, hapus, schedule notif
3. Audit Log — riwayat aksi semua user, ekspor Excel
4. Pengaturan KOP — upload logo pondok, alamat, nama channel
5. Backup/Restore data — fitur master setting (jika ada di app sekarang)
6. Generate API key, kalau ada

**Frekuensi check Sentry (saran):**

- Pagi (sebelum buka kelas): cek Issues unresolved 24h terakhir
- Sore (selesai mengajar): cek error spike kalau ada
- Setiap kali notif Sentry email/Slack masuk → langsung diagnosis

**Target stable kapan:**

Saran kyai: kumpulkan report 5-7 hari (1 minggu Internal Testing). Setelah itu evaluasi:
- Sentry error rate <0.1% per session = OK rilis
- Tidak ada critical bug (crash, data hilang, auth fail) = OK rilis
- Excel/PDF ekspor 100% jalan di semua role = OK rilis

Saat semua hijau → bilang ke saya "siap rilis production", saya akan otomatis revert config Opsi B → Opsi A, bump version 1.0.0 final, rebuild AAB, panduan kyai untuk upload Play Store production track.
