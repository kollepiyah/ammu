## Verifikasi behavior end-to-end (smoke test)

Test cepat di production untuk memastikan rules aktif:

1. **Test anonymous read** (harus tetap jalan):
   - Logout di app.
   - Open DevTools → Network → reload login page.
   - Cari Firestore `db_guru` request → harus return data (status 200).
2. **Test anonymous write** (harus diblokir):
   - Buka DevTools Console di anonymous tab.
   - Coba `firebase.firestore().collection('guru').add({nama:'test'})`.
   - Expected: error `permission-denied`. Kalau berhasil tulis → **RULES BELUM DEPLOYED**.
3. **Test signed-in write** (harus jalan):
   - Login normal.
   - Edit profil guru sendiri.
   - Save → harus sukses.

## Kontak panic button

Kalau write ke production gagal padahal user sudah login:

1. **Rollback rules** via Firebase Console → Rules → History → pilih versi sebelum v.108.44 → Publish.
2. Atau cepat:
   ```bash
   git show 82ef813^:firestore.rules > firestore.rules.rollback
   firebase deploy --only firestore:rules --project portal-mambaul-ulum
   ```
